## 2. v0.3 스키마 업그레이드 (실험 필드, 하위 호환)

QA 항목에 선택 필드 추가 (task_schema 갱신 + validator/verify 확장):
- `bundle_id` — 이 QA가 평가되는 materialized context bundle.
- `context_tier` — `32k|64k|128k|256k|512k`.
- `evidence_position` — `early|middle|late|multi`.
- `row_ids` — 표 QA에서 사용한 source 표의 행 ID(들).
- `gold_predicate` — 표/수치 QA의 결정론 필터·집계 술어(검증 재현용).

## 3. Source 계획

| source | 용도 | 라이선스 |
|---|---|---|
| 사용자 공고문 (신규) | long-context retrieval, 공고+법령 cross-document, eligibility | 공공기관 공고(§7/KOGL 가능) — 내부 취득, 공개는 locator만 |
| 법령 별표 HWP (가점제/과태료) **[파싱 입증됨]** | 진짜 table/numeric QA (가점 계산 등) | §7 비보호 |
| HUG/MOLIT 실데이터 (API 키) | table filtering/aggregation, 공고+표+법령 cross-source | 이용허락범위 제한 없음 |
| 기존 법령 3종 | cross-document legal, bundle distractor | §7 비보호 |

## 4. Context bundle materialize

- 32K/64K/128K/256K(가능시 512K) 토큰 번들을 구성: 타깃 근거문서 + 라이선스-안전 distractor(법령·공고·표 덤프) 연결, evidence 위치(early/middle/late/multi) 통제.
- 저장: `workspace_local/processed/bundles/{bundle_id}.txt` (내부 전용). 공개 QA는 `bundle_id`+`context_tier`+`evidence_position`만 참조.
- 토큰 카운트: tiktoken(설치) 또는 char 휴리스틱.

## 6. 검증 (v0.2 수준 엄격성 유지·확장)

- 표/수치: 파싱 셀·행에서 결정론 재계산(`gold_predicate` 재현).
- 공고/법령: grounding(gold_term verbatim) + 적대적 reviewer.
- bundle 무결성: 선언한 위치에 근거 존재, distractor 라이선스 확인.
- `validate_dataset.py`/`verify_qa.py` 새 필드 포함 확장.

## 7. 파서 타당성 (오늘 입증)

- 별표 HWP = HWP 5.0 OLE binary(`d0cf11e0`). `olefile`(설치됨)+`zlib`(raw-deflate)+레코드 파싱으로
  PARA_TEXT 추출 성공: `flSeq=48321121` → **[별표 1] 가점제 적용기준(제2조제14호 관련)**, 텍스트 2,692자·숫자 161개 복원.
- 배치 시 TABLE/CELL 레코드까지 파싱해 셀 그리드(점수표) 정밀 추출로 고도화.
- pip 설치 가능 확인(olefile 0.47) → tiktoken 등 추가 도구 설치 가능.

## 8. 실행 트리거

위 **1-A(키) + 1-B(공고 파일)** 가 들어오고 "go" 신호를 주시면, v0.3를 일괄 빌드:
Acquire(키로 실데이터 + 공고 파싱 + 별표 셀) → Extract(행/셀/공고 조항 + locator) →

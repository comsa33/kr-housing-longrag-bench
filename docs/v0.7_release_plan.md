anything — it states what is done and what remains before a `v0.7` research-preview release decision.

[docs/repository_scope_policy.md](repository_scope_policy.md); gates:
[docs/release_checklist.md §8](release_checklist.md#8-v07-research-preview-pre-release-baseline-scaffold--diagnostics).

## 3. Remaining before tagging v0.7

1. **Decision:** tag `v0.7` now as a research preview, OR run one more small (maintainer-approved) paid
   experiment first (e.g. a page-diverse answer remeasurement on a fixed smoke slice). Defer broad/full
   `test_public` and `dev` answer runs until the smoke protocol is stable.
2. If preparing the release, branch `feature/v07-release-packaging` work into release wording:
   - expand [docs/baseline_results_v07.md](baseline_results_v07.md) from a smoke draft toward a
     release-grade summary (the result cuts in [docs/v1_roadmap.md](v1_roadmap.md) §3) **without** new
     overclaiming;
   - keep `README.md` / `DATASET_CARD.md` agreeing that the public release is `v0.6.3` and v0.7 is a
     research preview.
3. Run the release gates in [docs/release_checklist.md](release_checklist.md)
   ([§1 automated gates](release_checklist.md#1-automated-gates) +
   [§8 v0.7 pre-release](release_checklist.md#8-v07-research-preview-pre-release-baseline-scaffold--diagnostics)).
4. Confirm no internal artifacts are tracked and no general/ranking claims leaked into public docs.
5. Only then decide the `v0.7` tag and any Hugging Face / Zenodo metadata updates — versions, citation,
   license, and caveats must agree across GitHub / HF / Zenodo before a release is called done.

## 4. Decision for the maintainer

> Tag `v0.7` now (research preview, smoke-level baselines documented), **or** keep collecting baseline
> evidence (one more approved paid experiment) before tagging?

Either path keeps `main` and `v0.6.3` untouched until the decision is made.

## 5. Non-claims

v0.7 is **not** leaderboard-ready, human-validated, sealed-hidden, a final model ranking, or paper-grade.
No hidden gold is published. Smoke results are illustrative, not benchmark tables.

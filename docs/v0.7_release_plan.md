## 3. Remaining steps (post-decision: tag v0.7 research-preview)

1. Merge this metadata PR (`feature/v07-release` → `develop`) after reviewer approval — release wording
   updated to `v0.7` research-preview; **no data change**, no new experiments, no paid runs.
2. With **explicit** maintainer approval, fast-forward / merge `develop` → `main` so the research-preview
   release lives on `main`.
3. Create the annotated tag `v0.7` on the correct `main` commit; create the GitHub release for `v0.7`
   (research-preview; baseline scaffold + smoke diagnostics; data unchanged at v0.6).
4. Only then update Hugging Face and Zenodo metadata **if** a v0.7 deposit is made — versions, citation,
   license, DOI state, and caveats must agree across GitHub / HF / Zenodo before the release is called
   done. If a v0.7 Zenodo deposit is minted, add its versioned DOI to `CITATION.cff` / `README.md` /
   `DATASET_CARD.md` (until then the concept DOI `10.5281/zenodo.20559127` is the citation DOI).
5. Re-run the release gates ([§1 automated gates](release_checklist.md#1-automated-gates) +
   [§8 v0.7 release](release_checklist.md#8-v07-research-preview-release-baseline-scaffold--diagnostics))
   on `main`, and confirm no internal artifacts are tracked and no general/ranking claims leaked.

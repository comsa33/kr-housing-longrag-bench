# v0.4-public Plan

Status date: 2026-06-05

v0.3-dev is valid for preliminary experiments, but not yet a public benchmark. v0.4-public should turn it into a reproducible, diverse, externally defensible benchmark package.

## Acceptance Criteria

| Area | Minimum v0.4-public target |
|---|---:|
| Official announcement documents | 8-12 distinct official LH/SH/GH/public announcements |
| Announcement providers | At least 1 provider, preferably 2+ |
| Regions | At least 4 시도 and 6 시군구 |
| Announcement years | At least 2024-2026 |
| QA count | 700+ verified QA |
| Hidden/public split | `dev`, `test`, and optionally `hidden` |
| Context tiers | 32k, 64k, 128k, 256k, 512k |
| Long-context/cross-document QA | At least 300 total |
| Table/numeric QA | At least 250 total |
| Cross-source QA | At least 120 total |
| Answerability QA | At least 80 total |
| Human review | Family-stratified random sample, 20 per family or 15% total |
| Reproducibility | Public reconstruction script can rebuild internal context from official URLs and local keys/files |
| Leakage | 0 secret/raw-full-context leakage in tracked files |

## Hard Rules

1. Do not release raw PDF/HWP, raw API responses, or bundle text unless a source-specific redistribution decision explicitly permits it.
2. Public QA may include short answers, locators, row IDs, page IDs, predicates, and source URLs.
3. For leaderboard-style evaluation, provide a reconstruction workflow:
   - public target manifest
   - acquisition script
   - extraction script
   - bundle builder
   - verifier
4. If external users cannot run the benchmark without private internal files, do not call it public-ready.
5. Keep v0.3 as `dev/seed`; v0.4 should add `test` and optionally `hidden`.

## Work Packages

### WP1. Source Expansion

Use `data/v0.4_announcement_targets_seed.jsonl` as the initial official-LH target list. Run:

```bash
python3 scripts/acquire_lh_announcements_from_manifest.py
```

Then inspect `workspace_local/audit/lh-announcements-v04/summary.json`.

Drop any target that:

- has no official 모집공고문 PDF/HWP,
- is only a pamphlet/form package,
- is not an official provider page,
- is too noisy for page-level extraction.

### WP2. Multi-Announcement Extraction

Generalize `scripts/extract_lh_announcement_text.py` from one PDF to all acquired v0.4 PDFs:

- output per announcement under `workspace_local/processed/lh-sale-announcements-v04/{announcement_id}/`
- produce `document_pages.jsonl`
- produce `numeric_facts.jsonl`
- produce `extraction_report.json`
- produce a merged index `workspace_local/audit/index_lh_v04.json`

### WP3. QA Expansion

Generate `data/qa_v0.4_candidates.jsonl`.

Target distribution:

| Family | Target |
|---|---:|
| long_context_retrieval | 150 |
| long_distance_retrieval | 100 |
| table_numeric_reasoning | 250 |
| cross_document_legal_reasoning | 120 |
| cross_source_aggregation | 120 |
| answerability_detection | 80 |
| format_robustness | 60 |
| multi_document_comparison | 60 |
| total | 940 |

Avoid near-duplicates. No single announcement should contribute more than 20% of non-table QA.

### WP4. Context Bundles

Build bundles with multiple announcements:

- `32k`, `64k`, `128k`, `256k`, `512k`
- position bands: `early`, `middle`, `late`, `multi`
- at least 3 bundle templates:
  - announcement-heavy
  - law-heavy
  - table-heavy

All bundle text remains under `workspace_local/processed/bundles-v04/`.

### WP5. Public Reconstruction Package

Add:

- `scripts/rebuild_v04_from_public_manifest.py`
- `docs/public_reconstruction.md`
- `docs/release_checklist.md`

The reconstruction package must explain:

- which public URLs are fetched,
- where local secrets are needed for APIs,
- which raw files are internal only,
- how to rebuild bundles,
- how to run validation/verification.

### WP6. Verification

Extend:

- `scripts/validate_dataset.py`
- `scripts/verify_qa.py`
- `scripts/check_public_release_readiness.py`

Required checks:

- QA schema
- source IDs known
- row/page IDs exist
- predicates recompute answers
- bundle IDs exist
- evidence position matches bundle manifest
- no raw corpus fields
- no secret leakage
- no answer/gold term over-length leakage
- no single-announcement dominance beyond threshold

## Stop Conditions

Stop and report if:

1. fewer than 6 official announcement PDFs can be extracted cleanly;
2. a source needs login or private permission;
3. the public reconstruction flow cannot be made deterministic;
4. automatic QA verification fails above 1%;
5. human review finds serious grounding issues in more than 5% of sampled QA.

# v0.6 Quality Report (realism · bundles · split/release)

Status date: 2026-06-05
This pass improves **paper/benchmark quality** of the v0.6 corpus (2,011 verified QA, 41 announcements,
10 providers) without adding sources. It naturalizes question phrasing (answers/predicates/evidence
unchanged), materializes multi-provider long-context bundles, and defines a release-safe split policy.

**Status: public-ready seed benchmark — NOT a leaderboard-ready / "hidden" benchmark yet.** Reasons:
human-review verdicts are not yet filled, and the test_hidden answers, while masked in the public
questions file, are present in-repo (internal) rather than served by a held-out leaderboard.

## 1. QA realism pass

- Input `data/qa_v0.5_candidates.jsonl` → output `data/qa_v0.6_realistic_candidates.jsonl`.
- Deterministic, template-based rewriting (no LLM free-generation). Every `qa_id`, `answer`,
  `answer_type`, `evidence`, `source_ids`, `page_ids`, `row_ids`, `table_ids`, `cell_ids`,
  `gold_predicate`, `evaluation`, `split` is **byte-identical** to the source (checked by
  `check_question_realism_v06.py`; 0 invariant changes). Only `question` changed; added
  `original_question`, `question_style`, `rewrite_rationale`.

Targets met: real_user+analyst ≥ 70% ✓; diagnostic_probe ≤ 15% ✓ (≤ 20% hard); cloze ≤ 15% ✓.
Position/passkey stress probes (all `long_distance_retrieval` + a hashed slice of `long_context_retrieval`)
are intentionally kept as `diagnostic_probe` cloze.

Before → after examples:

  - before: `… 다음 빈칸에 들어갈 값은? — "공공분양주택 757세대 (전용면적 ____ 296세대,"`

  - before: `… 자격요건 표에서 '생애최초 우선공급'의 '생애최초 특별공급 소득기준' 기준은 무엇인가?`
  - after: `… 공고의 신청자격 기준에서 '생애최초 우선공급'의 '생애최초 특별공급 소득기준' 기준은 어떻게 되는가?`

> Note: the cloze→natural conversion preserves the **same located value** as the answer (e.g. the value

## 8. Round-2 quality hardening (no human review required)

Human review remains **deferred** (resource-bound): the 470-item sample is *prepared, verdict pending* —
no human-validated claim is made anywhere.

**(a) Hidden split / eval harness.** `scripts/eval_harness_v06.py` scores predictions
(`--pred file.jsonl` of `{qa_id, prediction}`) against gold, with gold for `test_hidden` loaded from the
INTERNAL `workspace_local/audit/qa_v0.6_test_hidden_answers.jsonl` (public hidden file stays masked).
Per-metric scoring (exact_numbers / boolean_and_reason / contained-answer); reports plain **and
cluster-weighted** accuracy by split / task_type / question_style. `--self-test` (gold-as-prediction)
returns 100% across all cells (scorer + gold wiring verified). Hidden-split protocol: distribute
`data/qa_v0.6_test_hidden_questions.jsonl` (masked), keep answers private, score via this harness.

**(b) Near-duplicate clusters + cluster-weighted scoring.** `scripts/compute_near_dup_clusters_v06.py`
assigns `cluster_id` / `cluster_size` / `cluster_weight` (= 1/size) to every QA by a normalized question
template (title / quoted-anchor / number / 시·도·시군구 / date tokens removed). Result: **2,011 QA →
286 clusters** (169 singletons, 59%); cluster-weighted "effective" size = **286**. The harness's
cluster-weighted accuracy makes parametric families (e.g. the 522-item retrieval template, the 84-item
cross-source templates) count ~1 cluster, not N — so breadth, not repetition, drives the score.

**(c) real_user share.** Raised 10.4% → **30.1%** by deterministically re-voicing comparison
(provider/region/multi-document) and region+public-data (cross_source) questions into citizen phrasing
("각각 제시하라" → "각각 알려 주세요", "얼마인가?" → "얼마인가요?"); answers/predicates/evidence unchanged
(invariance check: 0 changes). Remaining analyst-heavy families (precise table/legal/format/retrieval)
are honestly left as `professional_analyst`.

**(d) bundle_count definition unified.** `check_public_release_readiness.py` now reports
`bundle_count: N (referenced by QA)` (distinct bundle_id actually used) and `bundles_available: M`
(distinct bundle_id across loaded manifests), plus `bundle_bearing_qa`. Current v0.6: **110 referenced /
194 available / 1,553 bundle-bearing QA** — no more ambiguous single number.
